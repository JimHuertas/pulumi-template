Configuración

Pulumi requiere credenciales de los servicios en la nube para administrar y aprovisionar recursos.


  • Debe utilizar una cuenta de usuario de IAM que tenga acceso programático con derechos para implementar y administrar los recursos gestionados a través de Pulumi.
  • Si ha instalado y configurado previamente la AWS CLI, Pulumi respetará y utilizará sus ajustes de configuración.
  • Si no tiene instalada la AWS CLI o planea utilizar Pulumi desde una canalización de CI/CD, recupere el ID de clave de acceso y la clave de acceso secreta y, a continuación, establezca las variables de entorno y en la estación de trabajo. 
    $ export AWS_ACCESS_KEY_ID="YOUR_ACCESS_KEY_ID"
    $ export AWS_SECRET_ACCESS_KEY="YOUR_SECRET_ACCESS_KEY"
    Copy
  • Pulumi puede autenticarse en Azure mediante una cuenta de usuario o entidad de servicio que tenga acceso mediante programación con derechos para implementar y administrar los recursos de Azure, se recomienda instalar la CLI de Azure y, a continuación, autorizar el acceso con una cuenta de usuario. 
    $ az login
    Copy
  • Debes usar una cuenta de usuario o servicio de IAM que tenga acceso programático con derechos para implementar y administrar tus recursos de Google Cloud. Se necesitará una cuenta de usuario de IAM con permisos que puedan crear y completar un depósito de Cloud Storage. Se recomienda instalar el SDK de Google Cloud y, luego, autorizar el acceso con una cuenta de usuario. Si no está configurado para interactuar con tu proyecto de Google Cloud, configúralo con el comando con el ID del proyecto: 
    $ gcloud config set project "YOUR_GCP_PROJECT_ID"
    Copy
  • A continuación, Pulumi requiere credenciales de aplicación predeterminadas para interactuar con tus recursos de Google Cloud, así que ejecuta el comando para obtener esas credenciales: 
    $ gcloud auth application-default login
    Copy


Una vez configuranda las credenciales del servicio cloud. Si es la primera vez que ejecuta comandos pulumi, es posible que se le pida que inicie sesión en Pulumi Cloud. Después de iniciar sesión, la CLI procederá a guiarlo a través de la creación de un nuevo proyecto.

  • Primero, se le pedirá un nombre y una descripción del proyecto. Pulse para aceptar los valores predeterminados o especificar nuevos valores. 
    This command will walk you through creating a new Pulumi project.
    Enter a value or leave blank to accept the (default), and press ENTER.
    Press ^C at any time to quit.

    project name: (quickstart)
    project description: (A minimal AWS Pulumi program)
    Created project 'quickstart'
  • A continuación, se le pedirá un nombre de stack. 
    Please enter your desired stack name.
    To create a stack in an organization, use the format / (e.g. `acmecorp/dev`).
    stack name: (dev)
    Created stack 'dev'
  • Por último, se le solicitarán algunos valores de configuración para el stack, por ejemplo la región.

Despliegue de Stack



Vamos a desplegar el stack:


$ pulumi up
Copy

Este comando evalúa el programa y determina las actualizaciones de recursos que debe realizar. En primer lugar, se muestra una vista previa que describe los cambios que se realizarán al ejecutar la actualización:


Previewing update (dev):
                
        Type                 Name            Plan
    +   pulumi:pulumi:Stack  quickstart-dev  create
    +   └─ aws:s3:Bucket     my-bucket       create

Resources:
    + 2 to create

Do you want to perform this update?
> yes
  no
  details

Una vez finalizada la previsualización, se le ofrecen tres opciones entre las que elegir. Elegir "details" le mostrará una rica diff de los cambios a realizar. Elegir "yes" creará su nuevo bucket en el servicio cloud. Elegir "no" devolverá a la pantalla de usuario sin realizar la operación de actualización.
